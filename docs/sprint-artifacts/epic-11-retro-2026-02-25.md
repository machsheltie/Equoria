## What Went Well

### 1. Clubs System Design (9/10)

**Decision:** Three-tab clubs page (Discipline / Breed / My Club) captures the key social dynamics:

- Discipline clubs: competitive alignment by how you play
- Breed clubs: identity alignment by what horses you have
- My Club: governance and elections (the "politics" of clubs)

**Result:** Rich social structure in a compact UI.

### 2. Pinned Thread Pattern (8/10)

**Feature:** Pinned threads appear at the top of each message board section with a 📌 indicator. Pinned threads can only be created by moderators (per game design) — so the "New Post" disabled state makes sense contextually.

### 3. Election Vote Bars (9/10)

**Feature:** The club elections UI shows candidates with visual vote progress bars. Even with mock data, this conveys the social dynamics of club governance clearly.

**Quote:** "The vote bars make it feel like something is actually happening in the clubs." — Alice

### 4. Unread Badge on Messages (9/10)

**Pattern:** Unread count displayed as a chip on the Inbox tab label. Message rows are bold for unread, normal for read. Standard and immediately understandable.

### 5. Community Hub as Entry Point (8/10)

**Design:** The `/community` hub page provides context before diving into sub-features. The 3 feature cards (Message Board, Clubs, Hall of Fame) give players a clear overview of what the community offers.

## What Didn't Go Well

### 1. All Interactive Elements Disabled (4/10)

**Problem:** Every action in Epic 11 is disabled — New Post, Join Club, Compose Message, Cast Vote. The pages look complete but do nothing when clicked.

**Context:** This is intentional (backend not yet wired) but creates a "dead" feeling when testing.

**Mitigation:** Each disabled button has a `title` tooltip explaining "Coming soon" or "Sign in required." But the UX could be better with a cleaner "Coming Soon" pattern.

**Recommendation:** When features are intentionally disabled, use a consistent "Soon" badge pattern rather than `disabled` attribute alone.

### 2. Mock Thread Data Quality (6/10)

**Problem:** Mock message board threads had placeholder titles ("Selling my champion mare...") that felt generic. The mock data didn't reflect the richness of an actual game community.

**Fix:** Updated mock data with more specific, game-relevant thread titles ("Best groom personality for show jumping?", "My Thoroughbred filly just won her first competition!"). Better immersion.

**Impact:** ~20 minutes of content writing for mock data. Worth it.

### 3. Community Hub Not Linked From World Hub (5/10)

**Problem:** The World Hub (`/world`) doesn't have a Community card. Community is a separate navigation item, not a world location. This creates a navigation inconsistency — World Hub covers services, but social features are accessed differently.

**Decision:** Kept Community as a top-level nav item rather than a World Hub location. The World Hub is for horse services; Community is for player interactions. Conceptually distinct.

**Note:** May need clearer wayfinding guidance for players discovering Community for the first time.

## Critical Decisions

### Decision 1: Community as Top-Level Nav, Not World Hub Location

**Approach:** Community (message board, clubs, messages) lives in the top nav alongside Home, World, Competitions.

**Rationale:** Social features serve a different purpose than service locations. Players use Community for player-to-player interaction, not horse management.

**Verdict:** ✅ Correct. Keeps the World Hub focused on horse services.

### Decision 2: All Actions Disabled in MVP UI

**Approach:** Build complete UI with all interactions disabled rather than building partial functionality.

**Rationale:** The full UI provides a preview of what Community will offer. Partial functionality (e.g., only reading, no posting) creates confusing half-states.

**Verdict:** ✅ Correct for the current phase. Players can see what's coming without being confused by partial features.

### Decision 3: Elections in My Club Tab

**Approach:** Club governance (elections) lives in the My Club tab rather than a separate route.

**Rationale:** Elections are a club activity, not a standalone feature. My Club tab groups all club-member-specific content.

**Verdict:** ✅ Correct. Reduces route complexity.
