## Retrospective

### What Went Well

**Inventory equip was the standout delivery of Epic 16.**

- JSONB-based pattern (`User.settings.inventory` + `Horse.tack`) clean and consistent with existing codebase patterns
- Seed-from-existing-field-on-first-GET approach elegant: first-time GET seeds inventory from `Horse.tack` if `settings.inventory` is empty
- `HorsePicker` modal pattern reusable for any horse-selection flow
- `fetchWithAuth` unwrapping convention maintained cleanly across `useInventory.ts`

**Onboarding wizard delivered on time** — 3-step structure gives new players a clear entry experience and the `OnboardingGuard` redirect mechanism is solid infrastructure.

**Art asset scaffolding complete** — Celestial Night horse silhouette SVG in `frontend/public/placeholder.svg` plus naming conventions documented. Art is owner-managed (solo dev + artist); this will ship when ready, independent of feature work.

### What Didn't Go Well

**Onboarding wizard depth is a genuine gap.**

The 3-step wizard (Welcome → Starter Kit → Ready) orients new players but does not guide them through the actual game mechanics step-by-step. A new player still has to self-discover: bank → buy horse → farrier → feed shop → vet → riders → grooms → training → competitions → equip tack. This is the core onboarding problem that remains unsolved.

**Z-index token system still unaddressed** — modals, bottom nav, toasts, and dropdowns all use hard-coded or inconsistent z-index values. Third carry-forward.

## Architecture Decision: Hybrid OnboardingSpotlight

During retrospective, team debated two approaches for the full onboarding tutorial:

**Charlie's approach (data-driven engine):**

- Step definitions in `User.settings.onboardingFlow` (JSONB array)
- Completion tracked per step server-side
- Dismissible, respects user agency
- Route-based and API-based completion triggers

**Alice's approach (visual spotlight):**

- Pulsing `--celestial-primary` ring on UI elements
- Immersive, visual, matches Celestial Night identity
- Anchored tooltip card with step message
- `[data-onboarding-target]` attribute bridge

**Hybrid decision (approved by Heirr):**

Combine both: Charlie's step engine + Alice's visual spotlight + Dana's `data-onboarding-target` attribute pattern.

### TypeScript Interface

```typescript
interface OnboardingStep {
  step: number;
  message: string;
  route: string;
  highlightTarget: string; // matches [data-onboarding-target] attribute value
  completionTrigger:
    | { type: 'route'; path: string }
    | { type: 'api'; endpoint: string }
    | { type: 'manual' };
}
```

### Planned Step Flow (10 steps)

1. Bank — collect weekly coin allowance (`data-onboarding-target="bank-claim-button"`)
2. Store — browse and buy first horse (`data-onboarding-target="store-buy-button"`)
3. World > Farrier — shoe the horse (`data-onboarding-target="farrier-book-button"`)
4. World > Feed Shop — select nutrition (`data-onboarding-target="feed-shop-purchase-button"`)
5. World > Veterinarian — health check + genetics (`data-onboarding-target="vet-book-button"`)
6. World > Riders — hire first rider (`data-onboarding-target="rider-hire-button"`)
7. World > Grooms — hire first groom (`data-onboarding-target="groom-hire-button"`)
8. World > Training Center — first training session (`data-onboarding-target="training-start-button"`)
9. Competitions — enter first competition (`data-onboarding-target="competition-enter-button"`)
10. Inventory — equip tack to horse (`data-onboarding-target="inventory-equip-button"`)

### Implementation Plan (Epic 17-1)

- `User.settings.onboardingFlow`: `OnboardingStep[]` — persisted in JSONB
- `OnboardingSpotlight` component: App-level, queries `[data-onboarding-target="${currentStep.highlightTarget}"]`, renders pulsing ring + anchored tooltip
- `POST /api/auth/advance-onboarding` — increments `settings.onboardingStep` (or marks step done)
- Route-based auto-advance: hook in router for `completionTrigger: { type: 'route' }`
- `completedOnboarding` flag set true after step 10 — replaces current 3-step wizard gate
