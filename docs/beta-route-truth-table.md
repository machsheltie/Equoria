## Status Semantics

- **beta-live:** Visible to beta users. Uses real authenticated account data. Advertised primary actions work through real APIs and honest empty/error states.
- Historical `beta-readonly` and `beta-hidden` labels are not accepted as completion states for active beta routes. If a route remains in scope, it is implemented as `beta-live`; if product removes a route from beta scope later, that decision must be recorded explicitly.

Routes are classified by real behavior, not visual polish or historical completion claims.

## Beta-Live Verification Status

The active beta policy is no hidden or read-only substitutes for in-scope routes. 21R-1 through 21R-3 corrected the critical beta path and proved it in Playwright:

- `/register`
- `/onboarding`
- `/stable`
- `/horses/:id`

The passing beta-critical suite proves register -> onboarding -> `POST /api/auth/advance-onboarding` -> persisted starter horse -> `/stable` render, plus navigation from `/stable` to `/horses/:id`.

Remaining route-specific verification belongs to later 21R hardening work, especially 21R-4 through 21R-6.
