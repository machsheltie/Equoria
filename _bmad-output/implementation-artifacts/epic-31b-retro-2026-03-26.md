## What Went Well

1. **Clean 3-story arc** — Score generation (31B-1), breeding inheritance (31B-2), and API endpoints (31B-3) each built cleanly on the previous. No backtracking or rework between stories.

2. **Reuse-first approach** — conformationService.mjs exports (`normalRandom`, `clampScore`, `calculateOverallConformation`, `CONFORMATION_REGIONS`) were reused across all 3 stories without duplication. The service module pattern from 31A scaled well.

3. **Statistical validation rigor** — 1000+ sample tests for distribution verification (31B-1) and inheritance regression-to-mean (31B-2). No flaky failures despite randomness — generous tolerances worked.

4. **All tests passed on first run** — Stories 31B-2 and 31B-3 had zero debugging cycles. Clean implementation from story specs to passing tests.

5. **Code review caught a real bug** — Box-Muller `Math.log(0) = -Infinity` edge case in 31B-1 was caught by review. Guard (`u1 === 0` -> `Number.EPSILON`) + `clampScore` NaN fallback added. This protects all downstream consumers.

6. **Zero Prisma migrations** — Entire epic used the existing `conformationScores` JSONB field. No schema changes, no test DB migration issues (the problem that bit us in 31A).

---

## What Didn't Go Well

1. **Nothing significant** — This epic was remarkably smooth. The well-defined story specs, clear formula from PRD-02, and established patterns from 31A eliminated most friction.

2. **Minor: Scalability TODO deferred** — `getConformationAnalysis` loads all same-breed horses into memory for percentile calculation. O(N) is fine for <1000 horses/breed but will need SQL `percentile_cont()` or caching if breeds grow large. Documented as TODO comment in code.

---

## Key Insights

1. **Story specs with exact formulas accelerate development** — The inheritance formula (`baseValue = (sire*0.5 + dam*0.5)*0.6 + breedMean*0.4`) was specified in the PRD. No ambiguity, no design decisions needed during implementation. Stories 31B-2 and 31B-3 both passed first run because of this.

2. **Service module pattern scales well** — `conformationService.mjs` went from 4 exports (31B-1) to being imported by 3 consumers (controller, routes, tests) without modification. Clean boundaries pay off.

3. **Edge case documentation in stories prevents bugs** — Stories explicitly listed edge cases (null parents, single horse in breed, legacy horses, missing overallConformation). Every edge case was implemented and tested without discovery during development.

4. **Middleware reuse eliminates boilerplate** — `requireOwnership('horse')` pre-fetching `req.horse` meant `getConformation()` needed zero DB queries. Pattern established in earlier epics continues to pay dividends.
